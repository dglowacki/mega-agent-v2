# WordPress SEO Guidelines

Comprehensive SEO best practices for WordPress content optimization.

## Content Quality

### Originality
- Create unique, original content
- Avoid duplicate content
- Add unique value to existing topics
- Provide fresh perspectives

### Comprehensiveness
- Cover topics thoroughly
- Answer related questions
- Include examples and use cases
- Address common objections

### Value to Reader
- Solve real problems
- Provide actionable advice
- Use clear explanations
- Include visual aids (images, diagrams)

### Regular Updates
- Review content quarterly
- Update statistics and examples
- Add new information
- Fix broken links

## Technical SEO

### Page Speed
- **Target**: < 3 seconds load time
- Optimize images (compress, lazy load)
- Minify CSS/JS
- Use caching
- Enable GZIP compression
- Use CDN for static assets

### Mobile-Friendly
- Responsive design
- Touch-friendly buttons (min 48x48px)
- Readable font sizes (16px minimum)
- No horizontal scrolling
- Fast mobile load time

### HTTPS
- Enable SSL certificate
- Redirect HTTP to HTTPS
- Update internal links to HTTPS
- Fix mixed content warnings

### Site Structure
- Clear navigation
- XML sitemap
- Robots.txt file
- Breadcrumb navigation
- Logical URL structure

### Structured Data
- Use Schema.org markup
- Article schema for posts
- BreadcrumbList schema
- Organization schema
- Local Business schema (if applicable)

## On-Page SEO

### URL Optimization
- Descriptive, keyword-rich URLs
- Use hyphens to separate words
- Keep URLs short (3-5 words)
- Lowercase letters only
- Avoid special characters

**Good**: `/react-tutorial-beginners/`
**Bad**: `/post?id=123&cat=tutorials`

### Title Tags
- **Length**: 50-60 characters
- Include primary keyword
- Front-load important keywords
- Unique for each page
- Compelling and click-worthy

**Formula**: Primary Keyword | Benefit | Brand

**Examples**:
- "React Tutorial for Beginners | Complete 2026 Guide"
- "WordPress SEO: 10 Tips to Rank Higher | Expert Guide"

### Meta Descriptions
- **Length**: 150-160 characters
- Include primary + secondary keywords
- Include call-to-action
- Accurately describe content
- Unique for each page

**Formula**: What + Benefit + CTA

**Example**: "Learn React with this comprehensive tutorial covering components, hooks, and state management. Perfect for JavaScript developers. Start learning today!"

### Headings (H1-H6)
- One H1 per page (page title)
- H2 for main sections
- H3 for subsections
- Include keywords naturally
- Maintain hierarchy (don't skip levels)
- Use descriptive headings

**Structure**:
```html
<h1>Main Page Title (Primary Keyword)</h1>
  <h2>Section 1 (Secondary Keyword)</h2>
    <h3>Subsection 1.1</h3>
    <h3>Subsection 1.2</h3>
  <h2>Section 2 (Secondary Keyword)</h2>
    <h3>Subsection 2.1</h3>
```

### Content Optimization
- **Length**: 1000-2000+ words for main content
- Include primary keyword in:
  - First 100 words
  - At least one H2
  - Meta description
  - URL
  - Image alt text
- Use synonyms and related terms
- Natural keyword density (1-2%)
- Answer user intent

### Images
- Descriptive file names (`react-tutorial-screenshot.jpg` not `IMG_1234.jpg`)
- Compress images (use tools like TinyPNG)
- Add alt text with keywords
- Use appropriate dimensions
- Consider WebP format
- Lazy load below-the-fold images

### Internal Linking
- Link to related content
- Use descriptive anchor text
- 2-3 internal links per 500 words
- Link to high-priority pages
- Update links when content changes

### External Linking
- Link to authoritative sources
- Use rel="nofollow" for untrusted sources
- Open external links in new tab (optional)
- Verify links regularly
- Avoid excessive external links

## Keyword Strategy

### Keyword Research
- Use tools: Google Keyword Planner, Ahrefs, SEMrush
- Focus on search intent
- Target long-tail keywords
- Consider keyword difficulty
- Analyze competitor keywords

### Keyword Placement
**Primary Keyword in**:
- Title tag
- Meta description
- H1 heading
- First paragraph
- At least one H2
- URL slug
- Image alt text
- Last paragraph

**Secondary Keywords**:
- Subheadings (H2, H3)
- Throughout content naturally
- Image file names
- Internal link anchor text

### Keyword Density
- **Target**: 1-2% for primary keyword
- Avoid keyword stuffing
- Use variations and synonyms
- Focus on natural language

### LSI Keywords
- Related terms and concepts
- Synonyms
- Common co-occurring words
- Topic-related phrases

**Example** for "React Tutorial":
- React components
- JavaScript framework
- Web development
- React hooks
- Frontend development

## Content Structure

### Introduction
- Hook reader in first sentence
- State what content covers
- Include primary keyword
- Promise value/benefit
- 100-150 words

### Body
- Use subheadings every 300-400 words
- Short paragraphs (3-5 sentences)
- Short sentences (15-20 words)
- Use bullet points and lists
- Include examples
- Add images/screenshots

### Conclusion
- Summarize key points
- Include call-to-action
- Link to related content
- Reinforce primary keyword
- 100-150 words

## Readability

### Flesch Reading Ease
- **Target**: 60-70 (Standard)
- Shorter sentences
- Simpler words
- Active voice
- Transition words

### Paragraph Length
- 3-5 sentences per paragraph
- Maximum 150 words
- One idea per paragraph
- White space between paragraphs

### Sentence Length
- Average 15-20 words
- Vary sentence length
- Break long sentences
- Use punctuation effectively

### Word Choice
- Simple, common words
- Avoid jargon (or explain it)
- Active voice preferred
- Strong verbs
- Concrete nouns

## Content Types

### Tutorial/How-To
- Step-by-step instructions
- Numbered lists
- Screenshots/images
- Expected outcome
- Common issues/solutions

### List Posts
- 7, 10, or 15 items
- Consistent structure
- Brief introduction to each
- Examples for each point
- Summary at end

### Reviews
- Pros and cons
- Feature comparison
- Use cases
- Rating/recommendation
- Personal experience

### News/Updates
- Timely content
- Original reporting
- Multiple sources
- Update date prominent
- Related coverage links

## Monitoring & Maintenance

### Track Metrics
- Organic traffic
- Keyword rankings
- Click-through rate (CTR)
- Bounce rate
- Time on page
- Conversions

### Regular Audits
- **Monthly**: Keyword rankings
- **Quarterly**: Content updates
- **Quarterly**: Broken link check
- **Annually**: Full SEO audit
- Continuous: Performance monitoring

### Update Content
- Add new information
- Update statistics
- Fix outdated examples
- Improve readability
- Add missing keywords
- Refresh meta descriptions

## Common Mistakes to Avoid

### Content
- ❌ Keyword stuffing
- ❌ Duplicate content
- ❌ Thin content (<300 words)
- ❌ Irrelevant content
- ❌ Outdated information

### Technical
- ❌ Slow page speed
- ❌ Not mobile-friendly
- ❌ Missing SSL certificate
- ❌ Broken links
- ❌ Missing alt text

### On-Page
- ❌ Missing title/description
- ❌ Non-descriptive URLs
- ❌ Missing headings
- ❌ No internal links
- ❌ Poor formatting
